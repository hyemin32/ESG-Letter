# 📬 ESG Letter — 매일 아침, ESG 뉴스가 알아서 정리되어 오는 뉴스레터

> 코딩을 전혀 모르는 사람이, AI와 대화하며 만든 **완전 무료·완전 자동** ESG 뉴스 브리핑 시스템입니다.
> 이 문서는 "이게 뭐고, 어떻게 작동하고, 어떻게 만들었는지"를 비개발자도 이해할 수 있게 처음부터 설명합니다.

---

## 1. 한 줄 소개

**평일 아침 7시, ESG 관련 최신 뉴스 5개를 AI가 골라·요약해서 웹페이지로 만들고, Slack으로 "오늘의 요약 + 링크"를 보내주는 시스템**입니다.
사람이 매일 뉴스를 검색하고 정리할 필요가 전혀 없습니다. 한 번 만들어두면 **매일 알아서** 돌아갑니다.

## 2. 왜 만들었나

ESG(환경·사회·지배구조) 분야는 뉴스가 많고 흐름이 빨라서, 매일 챙겨 읽기가 번거롭습니다.
그래서 이런 걸 원했습니다:

- 매일 **중요한 ESG 뉴스만** 추려서
- **핵심을 요약**해주고
- **이전에 다룬 이슈와 이어지면** 알려주고
- 어려운 용어는 **이해를 돕는 링크**까지 붙여서
- **폰 알림**으로 받고 싶다

이 모든 걸 **돈 한 푼 안 들이고** 구현했습니다.

## 3. 매일 어떤 일이 일어나나 (동작 흐름)

```
평일 오전 7시 (한국시간)
   │
   ├─ 1. ESG 뉴스 수집        (Google 뉴스에서 국내·해외 기사 100건+ 긁어오기)
   ├─ 2. 기사 5개 선별        (최신·중복제거·관련도로 국내 2 + 해외 3)
   ├─ 3. 본문 읽어오기        (각 기사 원문에 들어가 내용 추출)
   ├─ 4. AI 요약·정리         (Gemini AI가 요약 + 이전 이슈 연결 + 용어 추출)
   ├─ 5. 이해 돕는 링크 생성   (위키백과·검색·관련뉴스 — 지어낸 링크 없음)
   ├─ 6. 웹페이지 발행        (오늘자 뉴스레터 HTML 자동 생성·게시)
   └─ 7. Slack 알림 발송      (요약 + 웹페이지 링크를 폰으로 📲)
```

조용한 날(쓸 만한 뉴스가 적으면)에는 대신 **"ESG 용어 특별편"** 을 만들어, 중요한 용어 하나를 골라 쉽게 설명합니다. (이미 다룬 용어는 피해서, 시간이 지나면 나만의 ESG 용어사전이 쌓입니다.)

## 4. 어떤 부품들로 만들어졌나 (비유로 이해하기)

이 시스템은 **무료 서비스 여러 개를 레고처럼 조립**해서 만들었습니다.

| 부품 | 역할 | 비유 |
|------|------|------|
| **GitHub** | 코드 저장 + 전체 운영의 집 | 프로젝트가 사는 "집" |
| **GitHub Actions** | 매일 정해진 시간에 코드를 자동 실행 | 매일 아침 알아서 일어나는 "알람 + 일꾼" |
| **Google 뉴스 RSS** | ESG 뉴스 목록을 무료로 가져옴 | 신문 가판대 |
| **Gemini AI** | 기사를 읽고 요약·번역·용어 정리 | 똑똑한 "에디터 비서" |
| **GitHub Pages** | 뉴스레터를 웹페이지로 무료 게시 | 게시판 (영구 링크) |
| **Slack** | 완성된 요약을 폰으로 알림 | 집배원 📮 |

이 부품들이 매일 순서대로 손발을 맞춰 움직입니다.

## 5. 만들 때 필요했던 준비물 (한 번만 세팅)

코딩 없이, 아래 3가지 "열쇠"만 발급받아 안전한 금고(GitHub Secrets)에 넣으면 됩니다.

1. **GitHub 계정** — 무료. 코드와 자동실행, 웹페이지가 모두 여기서 돌아감.
2. **Gemini API 키** — Google AI Studio에서 무료 발급. AI 요약을 담당.
3. **Slack Incoming Webhook 주소** — Slack 앱에서 무료 생성. 알림을 받을 "우편함 주소".

> 🔒 이 열쇠들은 비밀번호와 같아서, 코드에 직접 적지 않고 **GitHub의 암호화된 금고(Secrets)** 에 넣습니다. 그래서 코드를 공개해도 안전합니다.

세팅 순서(정거장):
`① 코드를 GitHub에 올리기 → ② Gemini 키 발급·등록 → ③ Slack 웹훅 생성·등록 → ④ 웹 주소 변수 등록 → ⑤ GitHub Pages 켜기 → ⑥ 자동실행 쓰기권한 주기 → ⑦ 첫 발송`

## 6. 특별히 신경 쓴 점

- **이전 이슈와의 연결**: 최근 30일간 다룬 기사를 기억해두고, 오늘 뉴스가 그 후속이면 "지난 O/O에 다룬 △△의 후속" 이라고 알려줍니다.
- **가짜 링크 방지**: AI에게 "관련 링크 붙여줘"라고 하면 존재하지 않는 URL을 지어내기도 합니다. 그래서 **검색어만 AI가 만들고, 실제 링크(위키백과·구글 검색)는 프로그램이 생성**해 죽은 링크가 없습니다.
- **잘 멈추지 않게**: 느린 뉴스 사이트에서 무한 대기하지 않도록 **시간제한**을 걸어, 한두 기사가 실패해도 나머지로 뉴스레터를 완성합니다.
- **해외 뉴스도 한국어로**: 해외 영어 기사는 AI가 한국어로 번역·요약해서 실어줍니다.

## 7. 비용은?

**월 0원.** 모두 무료 범위 안에서 돌아갑니다.

- GitHub / GitHub Actions / GitHub Pages: 공개 저장소는 무료·무제한
- Google 뉴스 RSS: 무료
- Gemini AI: 무료 등급(하루 사용량 한도 내) — 하루 5건 요약은 한도의 극히 일부
- Slack: 무료 플랜

## 8. 만들면서 겪은 시행착오 (뒷이야기)

처음부터 매끄럽진 않았습니다. 비개발자가 AI와 대화하며 하나씩 해결했습니다.

- **"실행이 10분 넘게 안 끝나요"** → 느린 뉴스 사이트에서 무한 대기하던 것을 발견 → 본문 다운로드에 **8초 시간제한**, 링크 변환에 **12초 강제 제한**, 시도 횟수 상한을 추가해 30초 안에 끝나게 개선.
- **"AI 요약이 전부 실패해요 (429 오류)"** → 특정 AI 모델에 무료 사용량이 없어서 막힘 → 모델을 **항상 최신 무료 모델을 가리키는 별칭**으로 교체해 해결.
- **"모델이 없대요 (404)"** → 낡은 모델명을 최신으로 바꿔 최종 해결.

→ 에러 메시지를 한 줄씩 읽고 원인을 좁혀가며, 결국 **처음부터 끝까지 완전 자동화**에 성공했습니다.
